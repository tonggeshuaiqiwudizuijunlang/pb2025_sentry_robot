## 系统架构

### 数据流
```
海康相机 → armor_detector → armor_tracker → /tracker/target → standard_robot_pp_ros2 → 串口 → 下位机
```

### 关键节点
| 节点名 | 功能 | 发布话题 | 订阅话题 |
|--------|------|----------|----------|
| camera_node | 相机图像采集 | /image_raw | - |
| armor_detector | 装甲板检测 | /detector/armors, /detector/result_img | /image_raw |
| armor_tracker | 目标追踪 | /tracker/target | /detector/armors |
| standard_robot_pp_ros2 | 串口通信 | /serial/gimbal_joint_state | /tracker/target |

## 启动指令

### 完整自瞄系统启动
```bash
# 终端1：启动自瞄
source ~/pb2025_sentry_ws-main/install/setup.bash
ros2 launch rm_vision_bringup vision_bringup.launch.py
ros2 run rqt_image_view rqt_image_view
ros2 run rqt_reconfigure rqt_reconfigure
# 终端2：启动串口通信（如果不在vision_bringup中）
ros2 launch standard_robot_pp_ros2 standard_robot_pp_ros2.launch.py
```

### 单独启动各模块
```bash
# 单独启动相机
ros2 launch hik_camera hik_camera.launch.py

# 单独启动检测器
ros2 launch armor_detector armor_detector.launch.py

# 单独启动追踪器
ros2 launch armor_tracker armor_tracker.launch.py
```

### 编译指令
```bash
cd ~/pb2025_sentry_ws-main

# 编译自瞄相关包
colcon build --packages-select armor_detector armor_tracker hik_camera rm_vision_bringup

# 编译串口通信包
colcon build --packages-select standard_robot_pp_ros2

# 编译后source
source install/setup.bash
```

## 参数配置

### 配置文件位置
```
~/pb2025_sentry_ws-main/src/rm_shaobing_vision/rm_vision/rm_vision_bringup/config/node_params.yaml
```

### 关键参数说明

#### 相机参数
```yaml
/camera_node:
  ros__parameters:
    exposure_time: 3000    # 曝光时间(微秒)，自瞄推荐1000-5000
    gain: 8.0              # 增益
```

#### 检测器参数
```yaml
/armor_detector:
  ros__parameters:
    debug: true            # 开启调试图像
    detect_color: 1        # 0=红色, 1=蓝色
    binary_thres: 160      # 二值化阈值，推荐160-220
```

#### 串口参数
```yaml
/serial_driver:
  ros__parameters:
    device_name: /dev/ttyACM0
    baud_rate: 115200
```

### 动态参数调整
```bash
# 使用rqt_reconfigure
ros2 run rqt_reconfigure rqt_reconfigure

# 命令行设置曝光
ros2 param set /camera_node exposure_time 3000

# 命令行设置二值化阈值
ros2 param set /armor_detector binary_thres 160

# 命令行设置检测颜色
ros2 param set /armor_detector detect_color 1
```

## 常见问题解决

### 问题1：曝光参数错误
**错误信息**：
```
parameter 'exposure_time' has invalid type: Wrong parameter type
```

**解决方案**：
确保配置文件中exposure_time是整数格式（不带引号）：
```yaml
exposure_time: 3000    # 正确
# exposure_time: "3000"  # 错误
```

### 问题2：相机无图像
**排查步骤**：
```bash
# 1. 检查相机连接
lsusb | grep -i hik

# 2. 检查话题
ros2 topic hz /image_raw

# 3. 查看图像
ros2 run rqt_image_view rqt_image_view
```

### 问题3：识别不到装甲板
**排查步骤**：
1. 检查 `/detector/binary_img` 确认灯条是否被正确二值化
2. 调整 `binary_thres` 参数（降低到160-180）
3. 调整 `exposure_time`（降低到3000-5000）
4. 确认 `detect_color` 设置正确

### 问题4：QoS不匹配导致通信失败
**问题描述**：tracker发布的target消息无法被串口节点接收

**解决方案**：
修改 `standard_robot_pp_ros2.cpp` 中的订阅QoS：
```cpp
// 修改前
cmd_tracking_sub_ = this->create_subscription<auto_aim_interfaces::msg::Target>(
    "tracker/target", 10, ...);

// 修改后
cmd_tracking_sub_ = this->create_subscription<auto_aim_interfaces::msg::Target>(
    "tracker/target", rclcpp::SensorDataQoS(), ...);
```

## 无雷达单独测试自瞄

### 结论：可以单独测试，通信不受影响

### 原因分析

1. **自瞄系统独立性**
   - 自瞄系统（相机→检测→追踪→串口）与导航系统（雷达→SLAM→Nav2）是两个独立的模块
   - 它们共用同一个串口通信节点，但数据通道不同

2. **数据流分离**
   - 自瞄数据：`/tracker/target` → 串口 → 下位机云台控制
   - 导航数据：`/cmd_vel` → 串口 → 下位机底盘控制

3. **雷达不参与自瞄**
   - 自瞄使用PnP测距，不依赖雷达
   - 雷达仅用于导航建图和避障

### 单独测试自瞄的启动方式

```bash
# 方式1：只启动自瞄（推荐）
ros2 launch rm_vision_bringup vision_bringup.launch.py

# 方式2：如果串口不在vision_bringup中，需要单独启动
ros2 launch standard_robot_pp_ros2 standard_robot_pp_ros2.launch.py
```

### 注意事项

1. **TF树完整性**
   - 自瞄需要 `odom → gimbal_link → camera_optical_frame` 的TF变换
   - 这些TF由robot_state_publisher和串口节点提供，与雷达无关

2. **下位机配置**
   - 确保下位机正常发送IMU/云台姿态数据
   - 串口通信正常即可

3. **测试验证**
   ```bash
   # 检查TF是否正常
   ros2 run tf2_tools view_frames
   
   # 检查云台姿态数据
   ros2 topic echo /serial/gimbal_joint_state
   
   # 检查自瞄目标输出
   ros2 topic echo /tracker/target
   ```

### 可能的影响

| 项目 | 有雷达 | 无雷达 | 影响 |
|------|--------|--------|------|
| 装甲板检测 | ✓ | ✓ | 无影响 |
| 目标追踪 | ✓ | ✓ | 无影响 |
| 云台控制 | ✓ | ✓ | 无影响 |
| 底盘导航 | ✓ | ✗ | 无法导航 |
| 自动巡逻 | ✓ | ✗ | 无法巡逻 |

## 雷达安装方向调整

### 问题描述
雷达在物理世界中前后反装，导致建图方向错误。

### 配置文件位置
```
~/pb2025_sentry_ws-main/src/pb2025_robot_description/resource/xmacro/yq2026_shaobing_robot.sdf.xmacro
```

### pose 参数说明
```xml
<xmacro_block name="livox" ... pose="x y z roll pitch yaw" .../>
```
- `x y z`：位置偏移（米）
- `roll pitch yaw`：旋转角度（弧度）

### 当前配置
```xml
<xmacro_block name="livox" prefix="front_" parent="gimbal_yaw" 
    pose="0.0 0.0 0.35 ${pi} 0.0 ${pi}" update_rate="20" samples="1875"/>
```
- roll = π（180°）
- pitch = 0
- yaw = π（180°）

### 修改方法

**如果雷达前后反装**，需要修改 yaw 角（绕 Z 轴旋转 180°）：

| 当前 yaw | 修改后 yaw | 说明 |
|----------|------------|------|
| `${pi}` | `0` | 从 180° 改成 0° |
| `0` | `${pi}` | 从 0° 改成 180° |

**修改步骤**：

1. 编辑配置文件：
```bash
nano ~/pb2025_sentry_ws-main/src/pb2025_robot_description/resource/xmacro/yq2026_shaobing_robot.sdf.xmacro
```

2. 找到 livox 配置行，修改 yaw 值：
```xml
<!-- 原来（yaw = π） -->
<xmacro_block name="livox" prefix="front_" parent="gimbal_yaw" 
    pose="0.0 0.0 0.35 ${pi} 0.0 ${pi}" .../>

<!-- 修改后（yaw = 0） -->
<xmacro_block name="livox" prefix="front_" parent="gimbal_yaw" 
    pose="0.0 0.0 0.35 ${pi} 0.0 0" .../>
```

3. 重新编译：
```bash
cd ~/pb2025_sentry_ws-main
colcon build --packages-select pb2025_robot_description
source install/setup.bash
```

4. 重启导航系统验证。

### 其他安装方向问题

| 问题 | 修改参数 | 说明 |
|------|----------|------|
| 前后反装 | yaw ± π | 绕 Z 轴旋转 180° |
| 左右反装 | yaw ± π/2 | 绕 Z 轴旋转 90° |
| 上下反装 | roll ± π | 绕 X 轴旋转 180° |
| 倾斜角度不对 | pitch | 调整俯仰角 |

### 验证方法
```bash
# 启动导航后查看 TF
ros2 run tf2_tools view_frames

# 在 RViz 中查看点云方向是否正确
rviz2
# 添加 PointCloud2 显示雷达点云
```
